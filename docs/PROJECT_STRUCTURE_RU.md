# 📂 Структура проекта CollabCode AI

В данном документе приведено детальное описание архитектуры монорепозитория, чтобы жюри могло быстро сориентироваться в кодовой базе.

---

## 🔝 Общий вид (Root)

Проект организован по принципу монорепозитория для тесной интеграции типизации между клиентом и сервером.

- `/apps` — основные приложения (Web-клиент и API-сервер).
- `/packages` — общие библиотеки и типы.
- `/sandbox` — инфраструктура для изолированного запуска кода.
- `/docs` — расширенная техническая документация и ТЗ.
- `/demo` — демонстрационные наборы данных для холодного запуска.
- `docker-compose.yml` — оркестрация всех подсистем (БД, сервер, клиент, сандбокс).

---

## 🖥 `/apps/server` (Backend)

Express-сервер, отвечающий за логику комнат, оркестрацию контейнеров и интеграцию с ИИ.

- `src/index.ts` — точка входа, инициализация HTTP и WebSocket серверов.
- `src/config/` — конфигурация и валидация переменных окружения (Zod).
- `src/lib/` — вспомогательные утилиты (Auth, ID generation).
- `src/services/` — ключевое ядро системы:
  - `ai-service.ts` — логика взаимодействия с OpenRouter (LLM).
  - `notification-service.ts` — отправка уведомлений в Telegram и Discord.
  - `room-orchestrator.ts` — управление жизненным циклом Docker-контейнеров для запуска кода.
  - `room-store.ts` — управление состоянием комнат и snapshot-ами.
  - `platform-store.ts` — взаимодействие с базой данных (пользователи, роли, комнаты).
  - `sandbox-runner.ts` — логика исполнения кода внутри контейнеров.
- `src/scripts/` — утилиты для инициализации (DB init, Seed).

---

## 🎨 `/apps/web` (Frontend)

Современное SPA приложение на Next.js 15 с использованием App Router.

- `app/` — маршрутизация и серверные компоненты:
  - `auth/` — страницы регистрации и логина.
  - `cabinet/` — личный кабинет пользователя (управление комнатами).
  - `room/[roomId]/` — основная рабочая область редактора.
- `components/` — UI-компоненты:
  - `ui/` — дизайн-система (кнопки, индикаторы, 3D-сцены на Three.js).
  - `editor-pane.tsx` — интеграция Monaco Editor с Yjs.
  - `room-client.tsx` — логика синхронизации состояния комнаты.
- `lib/` — клиентские утилиты:
  - `collaboration-client.ts` — управление подключением к серверу и awareness-протоколом.
  - `platform-api.ts` — клиент для взаимодействия с REST API.

---

## 🐍 `/sandbox/python-runner`

Это сердце безопасности проекта.

- `Dockerfile` — легковесный образ на базе Alpine Python.
- Содержит минимальный набор библиотек и настроен для безопасного одноразового запуска скриптов с ограничением ресурсов.

---

## 📦 `/packages/shared`

Общие контракты данных, используемые на обеих сторонах (Frontend & Backend).

- `src/index.ts` — описание интерфейсов для:
  - Сообщений WebSocket.
  - Структур AI-ответов.
  - Моделей участников и комнат.
  - Событий платформы.

---

## 📊 БД (PostgreSQL)

Для долгосрочного хранения данных используется PostgreSQL (таблицы пользователей, метаданных комнат и прав доступа). Для MVP данные по живому редактированию удерживаются в оперативной памяти сервера для мгновенного отклика.
