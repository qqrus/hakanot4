# Демо-пакет (5-7 минут)

## Цель показа
Показать жюри, что платформа:
- поддерживает реальные комнаты и роли пользователей;
- визуализирует состояние комнаты в реальном времени;
- использует AI-ассистента и AI-навигатор;
- имеет геймификацию и внешние уведомления;
- работает устойчиво даже при деградации внешнего AI.

## Перед стартом (2-3 минуты подготовки)
1. Поднять проект:
```bash
docker compose up -d --build
```
2. Проверить:
- web: `http://localhost:3000`
- server: `http://localhost:4000/health`
3. Создать тестовые аккаунты:
```bash
npm run seed:accounts --workspace @collabcode/server
```
4. Войти минимум в 2 браузера/профиля:
- Владелец (`owner.demo@collabcode.local`)
- Редактор (`editor.demo@collabcode.local`)

## Сценарий показа (5-7 минут)
### 0:00-1:00 — Главная и контекст комнаты
1. Открыть главную.
2. Выбрать комнату в селекте.
3. Кратко пояснить 3 ключевые метрики:
- Индекс стабильности
- Индекс синхронизации
- Темп команды
4. Показать 3D-модель и подписи к элементам.

### 1:00-3:00 — Комната, роли и совместная работа
1. Открыть комнату владельцем.
2. Во втором окне открыть эту же комнату редактором.
3. Показать совместное редактирование, ленту событий, статус синхронизации.
4. Поменять роль третьего аккаунта на `viewer` и показать ограничение на редактирование/запуск.

### 3:00-4:30 — AI-ассистент и AI-навигатор
1. Внести правку в код и дождаться AI-анализа.
2. Показать блок подсказок и severity.
3. Задать вопрос в AI-навигаторе по цели комнаты.

### 4:30-5:30 — Runtime и уведомления
1. Запустить выполнение кода.
2. Показать статус в консоли и в индикаторах.
3. Нажать тест интеграций и показать уведомление в Telegram-группе.

### 5:30-7:00 — Геймификация и кабинет
1. Открыть кабинет.
2. Показать лидерборд, уровень, ранг, достижения.
3. Показать статус AI-провайдера и статус интеграций.

## Чеклист «что нажимать»
1. `Главная` → выбрать комнату.
2. `Войти в комнату`.
3. Изменить код (редактор).
4. Нажать `Запустить`.
5. Открыть блок `AI-навигатор` и отправить вопрос.
6. `Кабинет` → `Тест уведомлений`.
7. `Кабинет` → проверить лидерборд и достижения.

## План B (если внешний AI недоступен)
Если OpenRouter не отвечает:
- Платформа автоматически переходит в локальный режим (`source: mock`).
- Показ не останавливать: продемонстрировать, что:
1. совместное редактирование работает;
2. runtime и события продолжают работать;
3. лидерборд/геймификация продолжают считать;
4. Telegram уведомления отправляются;
5. в кабинете видно статус AI (внешний/локальный).

Фраза для жюри:
«Система спроектирована с graceful degradation: при отказе внешнего LLM пользовательский сценарий не падает, а продолжается в локальном аналитическом режиме».
